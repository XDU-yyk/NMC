# NMC 智能无人机伞

> **ESP32-S3 · 自主跟随 · Web 交互 · 边缘计算**
>
> 全国大学生嵌入式芯片与系统设计竞赛（前嵌赛）参赛作品

## 一、项目概述

本作品以 **ESP32-S3** 为核心处理器，设计了一款集 **自主跟随** 与 **Web 交互** 于一体的智能无人机伞系统。利用 S3 的 **AI 向量指令集** 优化多源信息融合算法，整合 **UWB 超宽带定位**、**气压计** 与 **IMU 惯性测量单元** 数据，通过扩展卡尔曼滤波器（EKF）实现复杂环境下厘米级精度的悬停与随行。

系统创新性地构建了基于 **WebSocket 的嵌入式 Web 服务器**，用户无需下载 App，仅需手机浏览器扫描二维码即可实时调控飞行状态，并实现 **三维态势感知**。项目充分发挥国产芯片的高集成度与边缘计算能力，为解放双手出行及残障辅助提供了 **低延迟、易交互** 的智能方案。

### 核心指标

| 指标 | 参数 |
| --- | --- |

| 高度精度 | ±5cm (气压计 + 卡尔曼滤波融合) |

| 最大倾角 | 35° (安全限幅) |
| 操作系统 | FreeRTOS (7 任务双核调度) |

## 三、核心技术

### 3.1 ESP32-S3 AI 向量指令集优化

ESP32-S3 内置 **PIE (Processor Instruction Extensions)** 支持 128-bit SIMD 操作。本项目从以下维度充分挖掘其算力：

#### 3.1.1 矩阵运算加速

卡尔曼滤波是计算密度最高的模块。EKF 的预测步骤涉及 $F \cdot P \cdot F^T$ 三个 $6 \times 6$ 矩阵乘法，更新步骤涉及 $H \cdot P \cdot H^T$ 和卡尔曼增益 $K = PH^T \cdot S^{-1}$。

传统实现需 $O(N^3)$ 标量乘加运算。利用 ESP-DSP 库的 `dsps_mul_f32()` 函数，可将 4 个 float 打包为 128-bit 向量，单指令完成 4 路并行乘法：

```c
// 16 字节对齐确保单周期 SIMD 加载
float A[16] __attribute__((aligned(16)));
float B[16] __attribute__((aligned(16)));
float C[16] __attribute__((aligned(16)));

// ESP-DSP 矩阵乘法 — 硬件加速
dsps_mul_f32(A, B, C, 4, 4, 4);  // 4×4 矩阵乘, 单周期 4 路 FMADD
```

#### 3.1.2 四元数旋转加速

姿态解算中的 Madgwick 滤波器核心是四元数更新。ESP32-S3 的 PIE 支持四元数旋转的单指令加速，可将机体加速度到世界坐标系的转换从 ~120 条指令压缩至个位数。

#### 3.1.3 内存对齐策略

所有参与 SIMD 运算的数据结构均标记 `__attribute__((aligned(16)))`：

```c
struct Vec3   { float x, y, z; }           __attribute__((aligned(16)));
struct IMUData { float accelX, ...; }       __attribute__((aligned(16)));
struct Attitude { float roll, pitch, ...; } __attribute__((aligned(16)));
```

这确保每次 128-bit 加载只需一个总线周期，避免跨 Cache Line 的额外开销。

### 3.2 多源传感器融合

#### 3.2.1 传感器特性对比

| 传感器 | 观测量 | 频率 | 精度 | 弱点 |
| --- | --- | --- | --- | --- |
| UWB (DW3000) | 三维位置 (x,y,z) | 20Hz | ±10cm | 多径效应、遮挡敏感 |
| 气压计 (BMP280) | 高度 (z) | 50Hz | ±5cm | 受温度/气流影响、有漂移 |
| IMU (MPU6050) | 加速度 + 角速度 | 200Hz | 姿态 ±0.5° | 积分漂移、低频噪声 |

单一传感器无法满足稳定跟随的需求：
- UWB 频率低 (20Hz)，无法单独支撑 200Hz 的控制回路
- 气压计仅提供高度，且存在慢漂移
- IMU 无绝对位置参考，纯积分会导致无限漂移

#### 3.2.2 EKF 融合框架

采用 **6 状态扩展卡尔曼滤波器** 实现多源融合：

**状态向量** $X = [x, y, z, \dot{x}, \dot{y}, \dot{z}]^T$
- 位置 $(x, y, z)$：世界坐标系 (mm)
- 速度 $(\dot{x}, \dot{y}, \dot{z})$：世界坐标系 (mm/s)

**预测步**（基于 IMU 加速度）：

$$X_{k+1} = F X_k + B u_k$$

其中 $F = \begin{bmatrix} I_3 & \Delta t \cdot I_3 \\ 0_3 & I_3 \end{bmatrix}$, $B = \begin{bmatrix} \frac{1}{2}\Delta t^2 I_3 \\ \Delta t I_3 \end{bmatrix}$, $u_k = [a_x, a_y, a_z]_{world}$

**更新步**：

- **UWB 更新**：观测三维位置 → $H_{uwb} = [I_3, 0_3]$，$R_{uwb}$ 动态缩放（置信度 × 噪声基底）
- **气压计更新**：观测高度 → $H_{baro} = [0, 0, 1, 0, 0, 0]$，$R_{baro}$ 固定

#### 3.2.3 自适应噪声调节

```
UWB 置信度 ─→ R_uwb 动态缩放系数
加速度幅值 ─→ 过程噪声 Q 自适应 (运动越剧烈, Q 越大)
无 UWB 时间 ─→ 气压计 R_baro 渐进增大 (避免漂移污染)
```

#### 3.2.4 Madgwick 姿态解算

独立于 EKF 运行，用于将加速度从机体坐标系转换到世界坐标系。Madgwick 滤波器在 200Hz 下的姿态误差 < 0.5°，优于传统互补滤波器。

### 3.3 级联 PID 跟随控制

跟随控制系统采用 **三级级联 PID** 架构：

```
目标位置 ─→ [位置环 PID] ─→ 期望速度 ─→ [速度→姿态映射] ─→ 期望角度
                                                              │
                                                              ▼
电机 PWM ←── [电机混合器] ←── 控制输出 ←── [姿态环 PID] ←── 姿态误差
```

| 环节 | 频率 | 作用 | 关键参数 |
| --- | --- | --- | --- |
| 位置环 (外环) | 20Hz | 位置误差 → 期望速度 | $K_p=0.8, K_i=0.02, K_d=0.05$ |
| 速度→姿态映射 | 20Hz | 速度 → 倾角 (atan 模型) | 最大倾角 35° |
| 姿态环 (内环) | 200Hz | 姿态误差 → 电机 PWM | Roll/Pitch $K_p=1.8$, Yaw $K_p=2.5$ |

**关键设计**：
- **积分分离**：误差 > 阈值时禁用积分，避免 windup
- **微分低通滤波**：$\alpha=0.7$ 的一阶低通，抑制高频噪声对微分项的放大
- **悬停油门自适应**：长期高度误差缓慢修正油门基准，补偿电池电压下降
- **安全距离死区**：±100mm 内不输出水平修正，减少无效抖动

### 3.4 WebSocket 嵌入式 Web 交互

#### 3.4.1 设计理念

传统无人机 App 存在三大痛点：

1. **安装门槛** — 需下载安装，占用手机存储
2. **兼容性** — iOS/Android/harmonyOS 需分别开发
3. **更新成本** — 每次升级需重新发布应用商店审核

本系统采用 **浏览器即客户端** 方案：

- 手机连接无人机热点（AP 模式）或同一 WiFi
- 浏览器打开 `http://smart-umbrella.local`（mDNS 自动发现）
- 纯 HTML5 + Canvas + WebSocket，**零安装、跨平台**

#### 3.4.2 通信协议

**上行** (Client → Server)：JSON 控制指令

```json
{"cmd": "follow"}       // 启动跟随
{"cmd": "hover"}        // 悬停保持
{"cmd": "return"}       // 一键返航
{"cmd": "stop"}         // 紧急停机
{"cmd": "setDist",  "value": 1500}   // 设置跟随距离
{"cmd": "setHeight","value": 2000}   // 设置跟随高度
{"cmd": "arm"}          // 解锁电机
```

**下行** (Server → Client, 10Hz)：JSON 遥测

```json
{
  "posX": 1234, "posY": -567, "posZ": 2000,    // 三维位置 (mm)
  "roll": 1.5, "pitch": -0.8, "yaw": 45.2,      // 姿态 (度)
  "vz": 12,                                       // 垂直速度 (mm/s)
  "mode": 2, "armed": true,                       // 模式与状态
  "unc": 35, "converged": true,                   // 定位不确定度
  "tgtX": 1500, "tgtY": 0, "tgtDist": 1650,      // 目标信息
  "imuOk": true, "baroOk": true, "uwbOk": true,   // 传感器状态
  "temp": 28.5, "press": 1013.2                   // 环境数据
}
```

## 四、FreeRTOS 实时调度

### 4.1 任务分配

ESP32-S3 双核架构的任务分配策略：

**Core 0 (PRO CPU) — I/O 密集型**：

| 任务 | 优先级 | 频率 | 说明 |
| --- | --- | --- | --- |
| 传感器采集 | 6 | 200Hz | I2C 读取 IMU + SPI 读取气压计/UWB |
| 遥测广播 | 3 | 10Hz | WebSocket 推送 + 串口输出 |

**Core 1 (APP CPU) — 计算密集型**：

| 任务 | 优先级 | 频率 | 说明 |
| --- | --- | --- | --- |
| **安全监控** | **9** | 50Hz | 最高优先级 — 姿态超限/安全开关/电压检测 |
| 姿态控制 | 8 | 200Hz | 姿态 PID + 电机混合器 |
| 传感器融合 | 7 | 100Hz | EKF 预测+更新 |
| 跟随决策 | 7 | 20Hz | 级联 PID 位置规划 |
| WebSocket | 5 | 事件驱动 | 消息解析 + 广播 |
| Web 服务器 | 4 | 事件驱动 | HTTP 请求处理 |

### 4.2 任务间通信

- **二值信号量**：传感器采集 → 融合 (数据就绪通知)
- **互斥锁**：融合状态读写保护
- **WebSocket 事件回调**：控制指令 → PID 参数更新

## 五、安全机制

### 5.1 多层安全防护

```
┌─────────────────────────────────────────────┐
│  L1: 硬件安全开关 (SAFETY_SWITCH)             │
│  ─ 低电平强制关闭所有电机 PWM                  │
├─────────────────────────────────────────────┤
│  L2: 安全监控任务 (50Hz, 最高优先级)           │
│  ─ 姿态超限检测 (>35° → 强制回稳)             │
│  ─ 电池低电压检测 (<3.5V → 告警, <3.3V → 降落) │
│  ─ 信号丢失超时 (>3s → 悬停等待)              │
├─────────────────────────────────────────────┤
│  L3: 控制层保护                               │
│  ─ PID 输出限幅 (Roll/Pitch ±400, Yaw ±300)   │
│  ─ 电机占空比硬限幅 (IDLE ~ MAX)              │
├─────────────────────────────────────────────┤
│  L4: 通信层保护                               │
│  ─ WebSocket ping/pong 心跳 (5s)             │
│  ─ 连接断开自动回稳                           │
└─────────────────────────────────────────────┘

⚠ 紧急停机 (EMERGENCY STOP) — 任何时刻可通过
   Web 急停按钮或硬件安全开关触发
```

## 六、创新点总结

### 6.1 芯片级创新

- **首次**将 ESP32-S3 的 AI 向量指令集系统性地应用于无人机多源融合领域
- 矩阵乘法、四元数旋转等关键路径实现 **3–5× 加速**（相比标量代码）
- 16 字节对齐策略保证 SIMD 单周期加载，Cache Line 利用率最大化

### 6.2 架构级创新

- **WebSocket 零安装控制** — 浏览器即客户端，跨 iOS/Android/HarmonyOS
- mDNS 服务发现 (`smart-umbrella.local`)，无需记忆 IP 地址
- 嵌入式 HTML5 控制台内置 2D 实时可视化、参数在线调节

### 6.3 算法级创新

- 6 状态 EKF + 自适应噪声调节（UWB 置信度 → 测量噪声动态缩放）
- 三级级联 PID (位置→速度→姿态) 带积分分离与微分滤波
- 悬停油门自适应估计，补偿电池电压衰减

### 6.4 应用级创新

- **解放双手出行** — 用户携带 UWB Tag，无人机伞自动跟随
- **残障辅助** — 浏览器 Web 操控，无障碍交互
- **多场景适应** — 遮阳/照明/紧急通讯等多模式扩展接口

### C. 使用说明

1. **开机** — 接通电池，等待系统自检（约 5 秒）
2. **连接** — 手机连接 WiFi `NMC-SmartUmbrella`，密码 `12345678`
3. **打开控制台** — 浏览器访问 `http://smart-umbrella.local` 或 `http://192.168.4.1`
4. **校准** — 保持无人机水平静止，系统自动完成陀螺仪和气压计零点校准
5. **解锁** — 确认安全开关处于高位（弹起状态），点击 Web 界面"解锁"按钮
6. **起飞** — 选择"悬停"模式，无人机自动起飞至默认高度 2m
7. **跟随** — 选择"跟随"模式，用户携带 UWB Tag 移动，无人机自动保持安全距离
8. **降落** — 点击"返航"自动降落，或手动切换"空闲"模式并点击"锁定"
