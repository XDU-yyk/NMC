# 智能无人机伞项目计划表（6月2日 - 6月30日）

## 项目定位

本项目以 ESP32-S3 + F4V3S PLUS 飞控为核心，完成一套“可安全演示”的智能无人机伞原型。F4 飞控负责姿态稳定、电机输出、遥控器接管与失控保护；ESP32-S3 负责传感器读取、上层跟随逻辑、WebSocket Web 交互、摄像头画面与参数调节。

> 计划原则：先保证安全可飞，再完成 Web 交互，最后逐步加入跟随能力。所有自主功能必须保留遥控器一键接管。

## 关键修正

| 原计划问题 | 调整后做法 |
| --- | --- |
| 简介写了 UWB，但器材清单没有 UWB 模块 | 当前版本不把 UWB 作为必做硬件；若后续补购 UWB，再作为增强定位模块接入 |
| GPS + 单点 ToF 难以稳定识别“人”的相对位置 | 必做目标改为“定高悬停、Web 监控、近距离测距辅助跟随演示”；人体自主跟随作为挑战项 |
| ESP32-S3 不适合直接承担底层飞控闭环 | 明确 F4V3S 负责姿态和电机闭环，ESP32-S3 只发送模式/速度/RC 辅助指令 |
| 30 天内同时完成飞控、感知、Web、图传、户外全自主跟随风险过高 | 时间线加入 MVP、降级路线、每日验收标准和安全门槛 |
| 计划末尾“关键并行路径”未写完 | 已补齐关键路径、风险、预算、验收标准和答辩交付物 |

## 模块划分

| 模块 | 名称 | 核心任务 | 依赖关系 | 优先级 |
| --- | --- | --- | --- | --- |
| M1 | 硬件平台搭建 | 组装、供电、接线、通电验证 | 无 | 必做 |
| M2 | 飞控基础系统 | F4V3S 调参、电机配置、遥控器对码、基础悬停 | M1 | 必做 |
| M3 | 感知与跟随系统 | GPS/ToF/气压计/IMU 数据读取、简单融合、跟随辅助 | M1+M2 通信 | 必做+挑战 |
| M4 | Web 交互系统 | WebSocket 服务、状态面板、参数调节、低帧率图传 | M1 | 必做 |
| M5 | 系统集成与测试 | 桌面联调、绑绳测试、户外演示、安全验证 | M2+M3+M4 | 必做 |

## 交付目标分级

| 等级 | 目标 | 必须达到的演示效果 |
| --- | --- | --- |
| MVP | 可控飞行 + Web 态势感知 | 遥控器可稳定悬停；手机浏览器查看姿态/高度/测距/电量；Web 可修改参数 |
| 标准目标 | 近距离辅助跟随 | 绑绳或低高度环境下，目标移动时无人机能产生正确方向的跟随响应 |
| 挑战目标 | 户外自主跟随演示 | 在开阔场地、低风速、遥控器随时接管条件下完成短距离随行 |

## 关键并行路径

```text
M1 硬件安全基线
  -> M2 手动可飞与失控保护
      -> M5 绑绳/户外飞行测试
  -> M3 传感器读取与融合
      -> M5 桌面闭环验证
  -> M4 WebSocket 与页面
      -> M5 演示与参数调节
```

关键路径是 M1 -> M2 -> M5。只要飞控不稳定，不进入自主跟随；Web 和感知可以并行做，但不能替代飞控安全验证。

## 风险与降级方案

| 风险 | 影响 | 降级方案 |
| --- | --- | --- |
| GPS 室外误差较大 | 无法精准跟随 | 只用于起飞点、围栏和状态显示；近距离演示以 ToF/手动辅助为主 |
| 单个 ToF 只能测前方距离，不能可靠识别人 | 跟随目标易丢失 | 演示“前向定距辅助跟随”；答辩说明后续接入 UWB/视觉识别 |
| OV2640 图传占用内存和带宽 | WebSocket 数据卡顿 | 图传降到 QVGA/5fps，或改为 MJPEG 独立 HTTP 通道 |
| 电机启动导致 ESP32 复位 | 系统不稳定 | 增加独立降压、共地检查、电源滤波和线束固定 |
| 自主控制振荡 | 飞行安全风险 | 限制最大角度/速度，先绑绳测试，遥控器随时切手动 |
| 6月底时间不足 | 完成度风险 | 保 MVP：手动可飞 + Web 态势感知 + 桌面跟随闭环 |

## 最终验收清单

| 编号 | 验收项 | 通过标准 |
| --- | --- | --- |
| A1 | 硬件通电安全 | 线束固定，无短路，电源不异常发热 |
| A2 | 飞控基础能力 | 遥控器可解锁、悬停、降落、失控保护 |
| A3 | Web 交互 | 手机无需 App，浏览器可看状态、改参数 |
| A4 | 传感器数据 | GPS/ToF/气压/姿态至少 5Hz 稳定刷新 |
| A5 | 跟随辅助 | 桌面或约束环境下控制方向正确，限幅有效 |
| A6 | 安全兜底 | 任意自主模式下遥控器能一键接管 |
| A7 | 答辩材料 | 视频、流程图、接线图、测试记录和预算齐全 |

## 答辩表达建议

- 重点强调“ESP32-S3 负责边缘侧 Web 交互和上层智能决策，F4 飞控负责底层实时姿态闭环”，架构更可信。
- 对“UWB”不要写成已实现，除非实际补充模块；可以作为后续增强方向。
- 对“高精度跟随”建议改成“近距离辅助跟随与态势感知原型”，把比赛演示风险降下来。
- 演示优先顺序：手动安全飞行 -> Web 实时面板 -> 参数调节 -> 桌面/绑绳跟随 -> 户外低速跟随。
