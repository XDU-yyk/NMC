# No-FC 保底演示脚本

> 适用于 F4V3S PLUS 飞控已损坏情景下的演示。
> 定位：ESP32-S3 伴随传感与 Web 演示平台 · 近距离辅助跟随与态势感知原型

---

## 演示前准备

### 硬件清单确认

- [ ] ESP32-S3-DevKitC-1 × 1（插 USB 供电/串口）
- [ ] VL53L1X ToF 模块 × 1（接线：SDA→GPIO4, SCL→GPIO5, XSHUT→GPIO10, VCC→3V3）
- [ ] NEO-M8N GPS 模块 × 1（接线：TX→GPIO18, RX→GPIO15, VCC→5V）
- [ ] OV2640 摄像头 × 1（接线见 config.h，独立固件）
- [ ] USB 数据线 × 1（烧录+串口）
- [ ] 手机/笔记本 × 1（WiFi 连接 Web 页面）

### 固件就绪

- [ ] `esp32-s3-web-mvp` 已烧录（ToF + GPS + Web 面板）
- [ ] `esp32-s3-ov5640-diag` 已编译，可随时烧录

### 物理环境

- [ ] 桌面空间足够展示 ESP32-S3 和传感器
- [ ] 若要演示室外 GPS，需靠窗或室外；室内说明 GPS 无 fix 属正常
- [ ] 准备一个物体（书本/手掌）用于 ToF 测距演示

---

## 演示流程（建议 5-8 分钟）

### Step 1 — 开场 + 安全声明（30s）

> "这是一个无人机伞的原型系统，采用 ESP32-S3 伴随计算机加 F4V3S PLUS 飞控的双处理器架构。"
> "当前飞控因硬件损坏暂停集成，今天展示的是 ESP32-S3 的传感器读取与 Web 交互能力。"

**硬件展示：** 指向 ESP32-S3（主演示板）、ToF（前向测距）、GPS（定位）、摄像头（预览）。

---

### Step 2 — Web 面板展示（1.5min）

1. ESP32-S3 通电，稍等 10 秒启动
2. 手机连接 WiFi `NMC-SmartUmbrella`，密码 `12345678`
3. 打开浏览器访问 `http://192.168.4.1`

**关键看点：**
```
[ ] 页面加载正常，标题显示"ESP32-S3 传感演示平台"
[ ] 顶部橙色警告条：飞行控制器已损坏
[ ] 飞控状态：⛔ 离线（已损坏）
[ ] 系统状态卡片：运行时间、堆内存、WiFi 客户端
```

---

### Step 3 — ToF 测距演示（1.5min）

1. 指向 Web 面板上的 **VL53L1X ToF** 卡片
2. 拿起手掌/书本，在 ToF 传感器前方 **20cm ~ 2m** 范围内移动

**关键看点：**
```
[ ] 距离数值实时变化（如 "450 mm"、"1200 mm"）
[ ] 有效数据状态为 "✅ 是"
[ ] 移开物体 → 状态变为 "❌ 否"（超量程）
[ ] 放回物体 → 恢复有效
```

**话术：** "VL53L1X 提供 4m 内毫米级测距，用于前向定距辅助。"

---

### Step 4 — GPS 状态展示（1min）

1. 指向 Web 面板上的 **GPS NEO-M8N** 卡片

**室内场景：**
```
[ ] 在线状态：❌ 否 或 ✅ 是（模块已供电）
[ ] 定位有效：❌ 否（室内无卫星信号）
[ ] 卫星数：0
```

**室外/靠窗场景：**
```
[ ] 在线状态：✅ 是
[ ] 定位有效：✅ 是
[ ] 卫星数：≥5
[ ] HDOP：≤2.0
[ ] 有经纬度坐标
```

**话术：** "GPS 提供室外定位参考，可实现 2.5m 精度。"

---

### Step 5 — 参数调节展示（1min）

1. 在浏览器中打开 `http://192.168.4.1/api/params`
2. 或用 `curl` 测试：

```powershell
# 读当前参数
Invoke-WebRequest http://192.168.4.1/api/params

# 修改参数（带白名单校验）
Invoke-WebRequest -Method POST http://192.168.4.1/api/params `
  -Body '{"webPollMs": 500}' `
  -ContentType "application/json"
```

**关键看点：**
```
[ ] 参数以 JSON 格式返回
[ ] POST 修改后串口日志有记录
[ ] 非法参数写入被拒绝
```

---

### Step 6 — 架构说明（1min）

> "原始架构中，F4V3S 飞控负责姿态稳定、电机输出、遥控器接管和失控保护。"
> "ESP32-S3 只负责传感器、Web 交互和未来的有限辅助指令，不直接控制电机。"
> "当前飞控已损坏，飞控集成暂停。替换飞控后通过 UART3 恢复 MSP 通信。"

**指向 Web 面板底部的"架构说明"卡片，逐条读。**

---

### Step 7 — 摄像头预览展示（可选，需切换固件，1min）

若评委想看摄像头：

```powershell
# 烧录摄像头固件
C:\Users\yyk\.platformio\penv\Scripts\platformio.exe run `
  -e esp32-s3-ov5640-diag -t upload --upload-port COM55
```

1. 连接 WiFi `NMC-Camera`
2. 浏览器打开 `http://192.168.4.1`
3. 打开 `/capture.jpg` 或 `/stream`

**话术：** "摄像头提供约 5 FPS、0.5-1s 延迟的 Web 预览，用于态势感知而非飞行闭环控制。"

---

### Step 8 — 收尾 + 恢复路线（30s）

> "当前 ESP32-S3 的传感器读取、Web 交互、参数调试能力均已验证。"
> "下一步：更换飞控硬件后，通过 UART3 恢复 MSP 通信，完成飞控数据集成。"
> "安全门槛不变：无桨测试 → 绑绳测试 → 手动悬停验证 → 才进入辅助跟随。"

## 常见问答

| 问题 | 回答 |
|------|------|
| 为什么不能飞？ | 飞控硬件损坏，正在等待替换。 |
| ESP32 能代替飞控吗？ | 不能。ESP32-S3 不做姿态闭环，不直接驱动电机。 |
| 这个原型能做什么？ | 传感器读取（ToF/GPS）、Web 遥测面板、参数调节、摄像头预览。 |
| 什么时候能飞起来？ | 替换飞控后，通过 UART3 恢复 MSP 通信，完成安全验证后。 |
| 成本多少？ | 约 624 元（含电池、遥控器、机架、飞控、ESP32、传感器）。 |

## 风险声明（评委必问）

> 本演示为 **无飞控保底版本**，不展示飞行能力。原始完整架构需要 F4V3S 飞控正常工作。
> 所有自主辅助功能（跟随、避障、RC override）在飞控替换并经过安全验证之前不会启用。
> 不声称当前具备高精度人体跟随、复杂户外自主飞行或 UWB 定位能力。
